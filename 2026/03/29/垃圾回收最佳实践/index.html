Unity 垃圾回收最佳实践

前言

在 Unity 游戏开发中，垃圾回收（Garbage Collection，简称 GC）是影响游戏性能的关键因素之一。当 GC 触发时，游戏主线程会被暂停，导致帧率下降、卡顿等问题。特别是在移动设备上，频繁的 GC 甚至会造成明显的卡顿感。

本文基于 Unity 官方文档，详细讲解 GC 产生的原因以及优化技巧，帮助你写出高性能的游戏代码。

1. 理解 Unity 的 GC 机制

1.1 什么是托管堆（Managed Heap）

在 C# 中，内存分为两种：

  • 栈（Stack）：存储值类型（int、float、struct 等），自动管理
  • 托管堆（Managed Heap）：存储引用类型（class、string、array 等），由 GC 管理

当你创建一个引用类型对象时，Unity 会在托管堆上分配内存。当这个对象不再被引用时，GC 会回收这块内存。

1.2 Unity 的 GC 特点

重要：Unity 的 GC 不是分代式的（Non-generational）

.NET 的 GC 是分代式的，可以高效处理大量小对象。但 Unity 使用的是 Boehm GC，它会扫描整个托管堆，这意味着：

  • GC 时间与托管堆大小成正比
  • 频繁创建小对象会导致频繁的 GC
  • GC 暂停时间可能很长

1.3 什么时候会触发 GC

  • 托管堆空间不足时
  • 手动调用 System.GC.Collect()（不推荐）
  • 场景加载时

7. 性能分析工具

7.1 Unity Profiler

使用 Unity Profiler 分析 GC：

  1. 打开 Window > Analysis > Profiler
  2. 选择 Memory 模块
  3. 查看 GC Alloc 列，找出分配内存的代码

7.2 查找装箱

在 Profiler 的 CPU 模块中，搜索以下方法名：

  • System.Int32::Box
  • System.Single::Box
  • System.Object::Box

7.3 使用 IL 查看器

使用 Rider、ReSharper 或 dotPeek 查看 IL 代码，搜索 box 指令来定位装箱。

8. 总结

优化优先级

  1. 避免在 Update 中分配内存（最高优先级）
  2. 使用对象池复用 GameObject 和集合
  3. 避免字符串拼接，特别是每帧的拼接
  4. 使用 for 循环替代 foreach 遍历 List 和 Dictionary
  5. 缓存组件引用和 API 调用结果
  6. 使用 CompareTag替代字符串比较
  7. 避免装箱，使用泛型集合

核心原则

在性能关键的代码中（Update、FixedUpdate、LateUpdate），尽量避免任何内存分配。

记住：GC 优化的目标不是完全消除 GC，而是减少 GC 的频率和影响。在合适的时机进行优化，避免过早优化。
